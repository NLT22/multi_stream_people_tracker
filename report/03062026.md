## 3. ReID Fine-tune

Config chính:

| Thành phần | Giá trị |
|------------|---------|
| Backbone | Swin-Tiny `swin_tiny_patch4_window7_224` |
| Input | RGB `256x128` |

| PK batch | P=24, K=4, batch=96 |
| Gradient accumulation | 2, effective batch=192 |
| Batches per epoch | 200 |
| Sampled crops / epoch | 19,200 |
| Output checkpoint | `output/reid_mmp/best.pth` |
| Output ONNX | `output/reid_mmp/swin_tiny_mmp_reid.onnx` |

Best validation result:

| Metric | Giá trị |
|--------|:-------:|
| Best epoch | 2 |
| Train gap | 0.595 |
| Val positive similarity | 0.520 |
| Val negative similarity | -0.033 |
| **Val gap** | **0.553** |

Training stopped at epoch 10:

| Epoch | Loss | Acc | Train gap | Val gap |
|------:|-----:|----:|----------:|--------:|
| 1 | 4.6910 | 0.214 | 0.607 | 0.485 |

## 4. Kết Luận Sử Dụng

| Thành phần | Đánh giá | Hành động |
|------------|----------|-----------|
| YOLO MMP | **Tốt, đáng dùng** | Dùng `models/yolov11/yolo11n_mmp.onnx` cho DeepStream |
| ReID MMP | **Khá, đáng test tracker** | Dùng ONNX để kiểm tra ID switch thực tế |
| MTMC/global ID | Chưa đánh giá | Cần chạy pipeline + metrics trên val scenes |

Config ReID gợi ý cho NvDeepSORT:

```yaml
ReID:
  reidFeatureSize: 256
  inferDims: [3, 256, 128]
  addFeatureNormalization: 0
  onnxFile: "output/reid_mmp/swin_tiny_mmp_reid.onnx"
  minMatchingScore4ReidSimilarity: 0.35
```

Không nên dùng threshold quá cao kiểu `0.78` của Market pretrained vì model MMP hiện có val positive mean khoảng `0.52`.

---

## 5. Việc Tiếp Theo

1. Tạo tracker config riêng `nvdeepsort_reid_swin_mmp.yaml`.
2. Chạy DeepStream trên một scene validation, ví dụ `lobby_3`.
3. Đánh giá per-camera MOTA/IDF1 và số ID switch.
4. Nếu ReID gây mất match, tune `minMatchingScore4ReidSimilarity` trong khoảng `0.30–0.45`.
5. Nếu cần cải thiện ReID, ưu tiên eval retrieval đầy đủ hoặc tăng `--batches-per-epoch`, không warm-start từ MTA làm mặc định vì khác domain.
