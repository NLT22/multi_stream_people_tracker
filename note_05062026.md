## Bối cảnh

Dự án đang **chuyển từ NvDeepSORT sang NvDCF** làm tracker chính. Hai vấn đề cần giải quyết song song:

1. **Throughput**: 12 cam đạt 16.5 FPS/cam nhưng 16 cam chỉ đạt 9.0 FPS (fail mục tiêu 10 FPS/cam). Bottleneck là compute (GPU 100%), không phải VRAM (1.7 GB / 16 GB). *Benchmark chạy trên RTX 5060 Ti — trên RTX 3050Ti 4 GB mốc sẽ thấp hơn.*
2. **Global IDF1**: Non-retail loanh quanh 0.75–0.80, retail bị kéo xuống ~0.40 do GT annotation chứa người đứng sau tủ kệ mà model không thể thấy → FN cố định, không thể sửa bằng tuning.

## Vấn đề 1 — Throughput: 20 cam × 10 FPS/cam

### Bước 0 — Chẩn đoán bottleneck trước

```bash
source venv/bin/activate
python scripts/benchmark_fps_ablation.py \
  --source dataset/MMPTracking_short/lobby_0/cam1.mp4 \
  --variants detector_only tracker_iou tracker_lite tracker_recall full_lite full_main \
  --cam-counts 4 8 12 16 20 \
  --target-fps 10 --stop-on-fail
```

Kết quả sẽ cho biết bottleneck chính là detector hay NvDCF tracker hay gallery probe.

### Biện pháp A — NvDCF compute reduction (đã implement)

`configs/tracker/nvdcf_perf_mmp_20cam.yaml` (mới tạo):
- `featureImgSizeLevel: 1` → 12×12 px/channel, tiết kiệm ~55% DCF compute so với level 2 (18×18)
- `useColorNames: 0` → bỏ 10 channel ColorNames
- `maxTargetsPerStream: 40` → MMPTracking indoor hiếm khi vượt 30 người/cam
- `maxShadowTrackingAge: 30` → giảm số target "ảo" đang được track

### Biện pháp B — Sub-batching (đã có sẵn)

`pipeline_mmp_realtime_20cam.yaml` đã có `sub_batches: "5:5:5:5"` — 4 NvDCF instances song song × 5 streams. `builder.py` đã truyền prop này vào nvtracker.

### Biện pháp C — Giảm resolution detector (đã implement)

`configs/models/nvinfer_yolov11_mmp_416_iv4.yml` + `configs/pipeline_mmp_realtime_20cam_416.yaml`:
- `infer-dims: 3;416;416` thay vì 640×640 → ~40% ít compute hơn
- TRT engine tự build lại lần đầu chạy (~2 phút)

```bash
# Test với 640px (hiện tại)
python -m src.main \
  --config configs/pipeline_mmp_realtime_20cam.yaml \

# Test với 416px nếu 640 không đủ
python -m src.main \
  --config configs/pipeline_mmp_realtime_20cam_416.yaml \
  --mmp-short-dataset dataset/MMPTracking_short:lobby_0
```

### Biện pháp D — Tăng inference interval lên iv6 (chưa làm)

## Vấn đề 2 — Global IDF1 >0.8

### 2A — Retail: loại phantom GT annotations (đã implement)

**Vấn đề gốc**: GT CSV chứa `person_id` cho người đứng sau tủ kệ — `visibility` tính từ bbox clipping không phát hiện được vì bbox vẫn nằm trong frame. Model không thể detect → FN cố định → Global IDF1 ~0.40.

**Workflow (cần chạy trên máy mới):**

```bash
# Bước 1: Export predictions cho retail scene
python -m src.main \
    --config configs/pipeline_mmp_nvdcf_realtime_baseline.yaml \
    --mmp-short-dataset dataset/MMPTracking_short:retail_0 \
    --no-display --no-sync \
    --export-predictions output/eval/mmp_retail_0

# Bước 2: Tìm phantom IDs (recall < 5%)
python scripts/detect_phantom_gt.py \
    --scene retail_0 \
    --pred-dir output/eval/mmp_retail_0
# → Ghi output/eval/mmp_retail_0/phantom_ids.txt

# Bước 3: Eval lại với clean GT
python -m src.eval.metrics_mmp \
    --short-root dataset/MMPTracking_short \
    --scene retail_0 \
    --pred-dir output/eval/mmp_retail_0 \
    --exclude-person-ids output/eval/mmp_retail_0/phantom_ids.txt
```

Chạy tương tự cho `retail_1` đến `retail_7`.

**Tham số của `detect_phantom_gt.py`:**
- `--recall-threshold 0.05` (mặc định): loại person_id được detect trong <5% số frame GT của nó
- `--iou-threshold 0.3`: ngưỡng match thấp (0.3 thay vì 0.5) để không bỏ sót detection mờ
- Có thể thử `--recall-threshold 0.10` nếu muốn loại thêm các ID bị khuất bán phần

### 2B — Non-retail: sweep nearline trên scenes yếu (chưa làm)

Mốc hiện tại (8 non-retail scenes, nearline): micro Global IDF1 = **0.7444**

Scene yếu nhất:

```bash
    --export-predictions output/eval/mmp_${SCENE}_nvdcf_realtime_baseline

```

**Bước 2**: Sweep nearline rộng hơn trên 3 scene yếu:

```bash
python scripts/sweep_nearline_mmp.py \
  --scenes cafe_shop_3 industry_safety_4 office_2 \
  --thresholds 0.55,0.58,0.62,0.65,0.70 \
  --margins 0.02,0.03,0.05 \
  --geo-weights 0.15,0.20,0.25,0.30,0.40 \
  --geo-min-overlaps 5,8,12 \
  --window-frames 100,125,150 \
  --out-root output/eval/nearline_sweep_weak_scenes
```

**Bước 3**: Nếu sweep không cải thiện đủ → thử tăng `tracklet_window` trong baseline config:

Sửa `configs/pipeline_mmp_nvdcf_realtime_baseline.yaml`:
```yaml
reid:
  tracklet_window: 10  # tăng từ 6 → 10 (nhiều embeddings hơn → average ổn hơn)
```

**Bước 4**: Với `cafe_shop` (nhiều người mặc đồ giống nhau) thử bật `ambiguous_match_rejection`:

